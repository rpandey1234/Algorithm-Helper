Linked Lists Overview

Linked lists store data similar to arrays, but do not have a fixed size. Each data object has a pointer to a second object. There are singly linked and doubly linked lists. Doubly linked lists make it possible to traverse a list efficiently in both directions.

Use of Linked Lists vs. Arrays

  • Disadvantage: Requires more memory space than arrays because a node must store the memory address of the next element in the list, while arrays live in a contiguous memory block.
  • Disadvantage: Less efficient at selecting a value at a particular index position.
  • Advantage: Insertion and deletion operation in constant time.
  • Advantage: It's easier to store data of different sizes. An array assumes each element must be the same type and same size.
  • Advantage: A linked list can grow organically. An array's size is fixed, so the number of elements must be known before hand.
  • Advantage: A linked list is better suited for multi-threaded operations. Since changes are local, only a few pointers are changed. Anytime the array size changes, it requires a global lock. .

To iterate through a linked list, use the idiomatic for loop:

for (cp = start; cp->link != NULL; cp = cp->link)
