Web Programming - Overview

What happens when you navigate to a URL in your browser:

  • The browser looks up the IP address for the domain name. The DNS lookup first checks if the DNS record is cached before recursively searching through namespaces.
  • The browser sends an HTTP request to the web server. The request may contain cookies (key-value pairs) which are used to identify the user.
  • The web server may respond with a re-direct, which the browser follows by sending another HTTP request.
  • The web server handles the GET or POST request, by looking at parameters.
  • The web server sends back an HTML response.
  • The browser begins rendering the HTML and CSS (it doesn't need to receive the entire request).
  • The browser may send additional requests to the server to retrieve images, JavaScript files, or other objects embedded in the HTML from the website's content delivery network (CDN).
  • The browser may send even more requests. These are asynchronous (AJAX) requests which can update portions of the webpage.

There are two submission methods for a form, GET and POST.

  • "GET" means that form data is to be encoded (by a browser) into a URL. GET should be used for retrieving data.
  • "POST"means that the form data is to appear within a message body. POST can be used with things like logging in, storing and updating data, or sending email.
